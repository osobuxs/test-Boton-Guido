POC Integraciones — Índice

Página simple para navegar a las pantallas de prueba de WhatsApp y Meta (Facebook + Instagram).
Recomendación: servilo por http(s) (Vercel / Live Server), no como file://.

WhatsApp

1) Pantalla principal (POC WhatsApp)

Botón “Conectar WhatsApp” → abre popup → recibe resultado vía postMessage → permite guardar integración.
Abrir wa-poc.html

2) Bridge WhatsApp (popup retorno)

La abre el backend al final del callback. Consulta /meta/oauth/poll, hace postMessage al opener y redirige.
(No la abras “a mano” salvo para debug)
Abrir wa-bridge.html (debug)

3) Links backend WhatsApp (debug rápido)

Endpoints útiles para probar el flujo de WhatsApp desde el navegador.
Abrir /backoffice/whatsapp/signup/config Abrir /meta/oauth/poll?state=...
Tip: primero pedí config para obtener un state, después probás poll.

4) Checklist WhatsApp

Si algo falla:
  • Popups habilitados
  • META_REDIRECT_URI == callback real del backend
  • El dialog usa redirect_uri igual al callback
  • El exchange usa el mismo redirect_uri
  • App Domains / Valid OAuth Redirect URIs bien
Meta (Facebook + Instagram)

5) Pantalla principal (POC Meta)

Botón “Conectar Meta” → abre popup → autoriza Facebook + Instagram → recibe resultado vía postMessage.
Abrir meta-oauth-poc.html

6) Bridge Meta (popup retorno)

La abre el backend al final del callback. Consulta /meta/oauth/poll, hace postMessage al opener y redirige.
(No la abras “a mano” salvo para debug)
Abrir meta-bridge.html (debug)

7) Links backend Meta (debug rápido)

Endpoints útiles para probar el flujo Meta unificado (Facebook + Instagram) desde el navegador.
Abrir /backoffice/meta/signup/config Abrir /meta/oauth/poll?state=...
Tip: el flujo Meta crea/actualiza internamente las integraciones de FACEBOOK e INSTAGRAM.

8) Checklist Meta

Si algo falla:
  • Popups habilitados
  • META_REDIRECT_URI == callback real del backend Meta
  • El login de Meta debe volver a /meta/oauth/callback
  • La app de Meta debe tener permisos aprobados / habilitados
  • BACKOFFICE_BASE_URL y BACKOFFICE_BRIDGE_PATH deben apuntar al meta-bridge.html

Si querés, después te lo dejo todavía más prolijo con tarjetas visuales y un selector de entorno.
